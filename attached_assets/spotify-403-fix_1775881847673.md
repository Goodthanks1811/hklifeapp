# Spotify 403 Fix — React Native Expo

## The Problem

You complete the Spotify auth flow, tap agree on the Spotify landing page, land back in your app — but hit a 403. This happens after the auth handshake, meaning the issue is in the **token exchange**, not the login itself.

There are four common culprits.

---

## Cause 1 — Redirect URI Mismatch (most common)

The redirect URI in your code must be **character-for-character identical** to what's registered in your Spotify Developer Dashboard — including trailing slashes, casing, and scheme.

**Check:** Go to [developer.spotify.com](https://developer.spotify.com) → your app → Edit Settings → Redirect URIs. Copy it exactly and compare to what your code sends.

Examples of mismatches that will cause a 403:

| Dashboard has | Code sends | Result |
|---|---|---|
| `myapp://callback` | `myapp://callback/` | ❌ 403 |
| `myapp://callback` | `myapp://Callback` | ❌ 403 |
| `myapp://callback` | `exp://192.168.1.1:8081/--/callback` | ❌ 403 |

---

## Cause 2 — Using Client Secret on Mobile (wrong flow)

Spotify requires **PKCE** for mobile apps — no `client_secret`. If your token exchange includes a `client_secret`, Spotify will reject it.

```javascript
// ❌ Wrong for mobile
body: {
  client_id: CLIENT_ID,
  client_secret: CLIENT_SECRET,  // this breaks it on mobile
  grant_type: 'authorization_code',
  code: code,
  redirect_uri: REDIRECT_URI,
}
```

The correct mobile token exchange uses a `code_verifier` instead:

```javascript
// ✅ Correct — PKCE, no client_secret
const response = await fetch('https://accounts.spotify.com/api/token', {
  method: 'POST',
  headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
  body: new URLSearchParams({
    grant_type: 'authorization_code',
    code: code,
    redirect_uri: REDIRECT_URI,
    client_id: CLIENT_ID,
    code_verifier: codeVerifier,  // generated during PKCE setup
  }).toString(),
});
```

### Full PKCE Flow

```javascript
import * as Crypto from 'expo-crypto';
import * as AuthSession from 'expo-auth-session';

// 1. Generate code verifier + challenge
const codeVerifier = generateRandomString(64);
const codeChallenge = await Crypto.digestStringAsync(
  Crypto.CryptoDigestAlgorithm.SHA256,
  codeVerifier,
  { encoding: Crypto.CryptoEncoding.BASE64 }
).then(hash =>
  hash.replace(/\+/g, '-').replace(/\//g, '_').replace(/=/g, '')
);

// 2. Auth request includes code_challenge
const authUrl =
  `https://accounts.spotify.com/authorize?` +
  `client_id=${CLIENT_ID}` +
  `&response_type=code` +
  `&redirect_uri=${encodeURIComponent(REDIRECT_URI)}` +
  `&code_challenge_method=S256` +
  `&code_challenge=${codeChallenge}` +
  `&scope=streaming%20user-read-email`;

// 3. Token exchange uses code_verifier (stored from step 1)
```

Install the required packages:

```bash
npx expo install expo-auth-session expo-crypto expo-web-browser
```

## Cause 3 — Wrong Redirect URI for Expo Auth Session

If you're using `expo-auth-session`, the redirect URI it generates is **not** your custom app scheme. It looks like:

```
https://auth.expo.io/@your-username/your-app-slug
```

You need to register **that exact URI** in your Spotify dashboard, not `myapp://callback`.

Find yours with:

```javascript
import * as AuthSession from 'expo-auth-session';
console.log(AuthSession.makeRedirectUri({ useProxy: true }));
```

---

## Cause 4 — Account Not Whitelisted (Development Mode)

If your Spotify app is still in **Development Mode**, only accounts you've manually added can authenticate. Anyone else — including yourself on a secondary account — will be blocked.

**Fix:** Spotify Dashboard → your app → Users and Access → add your Spotify account email.

To remove this restriction entirely, apply for **Extended Quota Mode** in the dashboard.

---

## Checklist

| Check | What to look for |
|---|---|
| Redirect URI exact match | Compare dashboard vs code character by character |
| No `client_secret` in token exchange | Mobile must use `code_verifier` via PKCE |
| Expo redirect URI format | If using `expo-auth-session`, register the `auth.expo.io` URI |
| Account whitelisted | Dev mode only allows manually added users |
